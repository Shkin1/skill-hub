# 场景化分析模板

## 模板1：创业决策模板

### 适用场景
- 创业方向选择
- 市场进入决策
- 资源分配

### 核心问题清单
1. 主要矛盾是什么？（市场机会 vs 自身能力？）
2. 我的"井冈山"在哪里？（根据地）
3. 当前是战略防御、相持还是反攻？
4. 谁是我的朋友？谁是我的敌人？（统一战线）
5. 我是否做了充分的调查研究？

### 分析步骤
```
1. 列出所有可选方向
2. 用矛盾分析法找到主要机会
3. 判断自身实力阶段
4. 选择根据地（细分场景）
5. 制定持久计划
6. 设计验证机制
```

### 输出格式
```markdown
## 创业决策分析

### 主要矛盾
[哪个机会是主要矛盾？]

### 实力阶段判断
- 当前阶段：[防御/相持/反攻]
- 判断依据：[为什么是这个阶段]

### 根据地选择
- 领域：[哪里是细分市场]
- 理由：[为什么选这个领域]

### 持久计划
- 防御期：[做什么]
- 相持期：[做什么]
- 反攻期：[做什么]

### 验证机制
- 验证指标：[如何判断成功]
- 调整条件：[什么情况下调整]
```

### 常见错误
- ❌ 不调查就判断机会（主观主义）
- ❌ 在巨头强项上硬拼（冒险主义）
- ❌ 不选择根据地，全面铺开（分散兵力）
- ❌ 急于求成，不制定持久计划（速胜论）

## 模板2：团队管理模板

### 适用场景
- 团队冲突处理
- 激励机制设计
- 组织变革

### 核心问题清单
1. 团队的主要矛盾是什么？（目标不一致？资源分配？沟通不畅？）
2. 这是人民内部矛盾还是敌我矛盾？
3. 如何建立统一战线？（共同目标是什么？）
4. 如何从群众中来，到群众中去？（如何收集和执行意见）
5. 如何设计试点验证机制？

### 分析步骤
```
1. 调查研究（深入一线了解真实情况）
2. 分清矛盾性质（人民内部矛盾用民主方法）
3. 找到共同利益点（统一战线基础）
4. 设计解决方案（集中提炼）
5. 小范围试点（验证）
6. 全面推广（在推广中修正）
```

### 输出格式
```markdown
## 团队管理分析

### 矛盾分析
- 主要矛盾：[什么是最核心的问题]
- 矛盾性质：[人民内部矛盾/其他]

### 统一战线
- 共同目标：[什么可以团结大家]
- 团结策略：[如何团结]

### 解决方案
[系统化的方案]

### 试点计划
- 试点范围：[在哪里试点]
- 验证指标：[如何判断成功]

### 推广计划
[如何全面推广]
```

### 常见错误
- ❌ 不调研就做决策（官僚主义）
- ❌ 用强制方法处理人民内部矛盾（简单粗暴）
- ❌ 不试点就全面推广（冒险主义）
- ❌ 推广后不修正（僵化执行）

## 模板3：市场竞争模板

### 适用场景
- 竞争策略制定
- 市场定位
- 差异化竞争

### 核心问题清单
1. 对手的主要矛盾是什么？（他们的弱点在哪里？）
2. 力量对比如何？（敌强我弱？势均力敌？我强敌弱？）
3. 当前竞争阶段是什么？（防御？相持？反攻？）
4. 哪里是对手的"城市"？哪里是"农村"？
5. 如何集中优势兵力打歼灭战？

### 分析步骤
```
1. 力量对比分析（客观评估）
2. 判断竞争阶段
3. 选择战略路径（正面/侧翼）
4. 寻找根据地（对手忽视的领域）
5. 制定战术清单
6. 设置退出条件
```

### 输出格式
```markdown
## 市场竞争分析

### 力量对比
| 维度 | 我方 | 对手 | 判断 |
|------|------|------|------|

### 竞争阶段
- 当前阶段：[防御/相持/反攻]
- 判断依据：[为什么]

### 战略路径
- 路径选择：[正面/侧翼]
- 理由：[为什么]

### 根据地
- 位置：[哪里是对手忽视的]
- 可行性：[为什么可行]

### 战术清单
1. [战术1]
2. [战术2]

### 退出条件
- 条件1：[什么情况下调整]
- 条件2：[什么情况下退出]
```

### 常见错误
- ❌ 高估自己，低估对手（主观主义）
- ❌ 在敌强我弱时硬拼（冒险主义）
- ❌ 不在关键点集中资源（分散兵力）
- ❌ 不设置退出条件（僵化执行）

## 模板4：个人成长模板

### 适用场景
- 职业规划
- 能力提升
- 个人品牌建设

### 核心问题清单
1. 我的主要矛盾是什么？（能力不足？方向不明？资源不够？）
2. 我当前处于什么阶段？（防御？相持？反攻？）
3. 我的"井冈山"在哪里？（哪个细分领域可以深耕？）
4. 如何制定持久计划？
5. 如何实践验证？

### 分析步骤
```
1. 自我分析（优劣势评估）
2. 阶段判断
3. 根据地选择
4. 制定持久计划
5. 设计实践验证
6. 设置调整机制
```

### 输出格式
```markdown
## 个人成长规划

### 自我分析
| 维度 | 优势 | 劣势 | 判断 |
|------|------|------|------|

### 阶段判断
- 当前阶段：[防御/相持/反攻]
- 判断依据：[为什么]

### 根据地选择
- 领域：[哪里是我的"井冈山"]
- 理由：[为什么选这个领域]

### 成长路线图
- 防御期（[时间]）：[做什么]
- 相持期（[时间]）：[做什么]
- 反攻期（[时间]）：[做什么]

### 实践验证
- 实践项目：[做什么项目验证]
- 验证标准：[如何判断成功]

### 调整机制
- 调整频率：[多久调整一次]
- 调整依据：[根据什么调整]
```

### 常见错误
- ❌ 不客观评估自己（主观主义）
- ❌ 阶段判断错误（速胜论/保守主义）
- ❌ 根据地选择错误（分散精力）
- ❌ 不实践只学习（教条主义）

## 模板5：冲突处理模板

### 适用场景
- 处理分歧
- 意见不一致
- 利益冲突

### 核心问题清单
1. 矛盾的性质是什么？（原则性/非原则性？暂时性/长期性？）
2. 主要矛盾是什么？（表面分歧背后的真正问题是什么？）
3. 共同利益点在哪里？（统一战线基础）
4. 哪里可以团结？哪里必须斗争？
5. 如何让实践检验代替争论？

### 分析步骤
```
1. 暂停争论，先调查
2. 分清矛盾性质
3. 找到共同利益点
4. 确定团结和斗争的边界
5. 设计验证方案
6. 复盘总结
```

### 输出格式
```markdown
## 冲突处理分析

### 调查事实
- 我的信息：[我掌握的信息]
- 对方的信息：[对方掌握的信息]
- 缺失的信息：[双方都不知道的信息]

### 矛盾性质
- 原则性/非原则性：[判断]
- 暂时性/长期性：[判断]

### 共同利益
- 一致目标：[什么是一致的]
- 分歧点：[哪里不一致]

### 处理原则
- 团结部分：[哪里可以妥协]
- 斗争部分：[哪里必须坚持]

### 验证方案
- 验证方法：[如何验证]
- 验证标准：[什么算成功]

### 复盘要点
- 成功因素：[什么做对了]
- 改进空间：[什么可以做得更好]
```

### 常见错误
- ❌ 把分歧当成敌对（扩大矛盾）
- ❌ 不调查就下结论（主观主义）
- ❌ 要么妥协要么对抗（极端化）
- ❌ 不复盘（重复犯错）
