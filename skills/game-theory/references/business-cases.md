# 博弈论商业案例库

本文档提供典型商业场景的博弈论分析模板，帮助快速映射实际问题到博弈框架。

---

## 案例 1：价格竞争（可口可乐 vs 百事可乐）

### 局势描述
两家饮料巨头在定价上博弈：
- 双方都高价 → 市场份额不变，利润都高
- 一方低价 → 抢夺市场份额，利润可能更高
- 双方低价 → 市场份额不变，利润都低

### 博弈结构
```
        | 可口高价 | 可口低价
百事高价 | 100,100 | 40,120
百事低价 | 120,40  | 70,70
```
（数字为利润，单位：亿）

### 分析
- 优势策略分析：低价对每方都是优势策略（100→120, 40→70）
- 纳什均衡：(低价,低价)，收益(70,70)
- 帕累托最优：(高价,高价)，收益(100,100)
- 结论：典型的囚徒困境结构

### 破解方法
1. **差异化竞争**：改变产品，减少直接价格博弈
2. **重复博弈**：长期竞争关系，价格战损害未来合作
3. **市场分割**：各自专注不同渠道/地区，减少正面冲突
4. **行业自律**：建立价格联盟（但法律限制）

## 案例 2：市场进入（网约车新玩家）

### 局势描述
某城市已有滴滴主导市场，新玩家考虑是否进入：
- 进入 → 滴滴可选择接受（分享市场）或打击（价格战）
- 不进入 → 维持现状

### 博弈树
```
新玩家 → 进入 → 滴滴 → 接受 → 新玩家收益30，滴滴收益50
                        → 打击 → 新玩家收益-10，滴滴收益20
       → 不进入 → 新玩家收益0，滴滴收益80
```

### 分析（后向归纳）
1. 滴滴决策点：打击收益20 vs 接受收益50 → 滴滴选择接受
2. 新玩家预期：滴滴会接受 → 进入收益30 > 不进入收益0
3. 均衡：新玩家进入，滴滴接受

### 打击可信性问题
如果滴滴有"打击承诺机制"（如预先投入价格战资源）：
- 打击成本已 sunk → 打击收益可能高于接受
- 新玩家预期会被打 → 不进入
- 滴滴独占市场，收益80

### 应用启示
- 进入者要评估在位者打击成本 vs 接受损失
- 在位者建立打击可信性可阻止进入
- 但打击承诺可能是过度投资（降低实际收益）

## 案例 3：谈判博弈（供应商与品牌方）

### 局势描述
品牌方要求供应商降价10%，供应商可选择：
- 接受 → 维持合作关系，利润降低
- 拒绝 → 可能失去订单，但保留利润率

### 信息不对称
- 品牌：知道供应商成本吗？（不确定）
- 供应商：知道品牌方其他供应商选项吗？（不确定）

### 最后通牒博弈结构
```
品牌方提出分配 → 供应商接受/拒绝
拒绝 → 双方收益为0
```

### 分析
- 理论最优报价：让供应商"勉强接受"（利润最小但非零）
- 但现实中，拒绝率随分配不公程度上升
- 供应商可能因"不公平感"拒绝不利条款

### 策略建议
| 角色 | 策略 |
|------|------|
| 品牌方 | 多议题打包（降价换订单量、付款周期优化）；逐步逼近底线而非极端报价 |
| 供应商 | 展示成本信息增加透明度；提供替代方案（其他价值交换）；建立"不能更低"的边界信号 |

## 案例 4：团队激励（销售团队）

### 局势描述
销售团队共10人，总奖金池100万：
- 业绩绑定：每人业绩贡献 → 按比例分配
- 个人业绩：努力工作 → 高业绩，但成本（时间、精力）
- 偷懒：低业绩 → 低成本，但收益可能降低

### 囚徒困境结构
- 个人偷懒收益 > 个人努力收益（如果团队中足够多人努力）
- 每人都偷懒 → 总业绩低 → 奖金池小 → 每人收益低

### 收益矩阵（简化版，两人团队）
```
        | 同事努力 | 同事偷懒
我努力 | 50,50    | 20,40
我偷懒 | 40,20    | 30,30
```
- 同事努力时：我偷懒(40) > 努力(20) → 倾向偷懒
- 同事偷懒时：我偷懒(30) > 努力(20) → 倾向偷懒
- 均衡：(偷懒,偷懒)，收益(30,30) ← 团队效率低

### 破解策略
| 机制 | 博弈论原理 | 设计要点 |
|------|------------|----------|
| 个人业绩排名奖 | 引入相对竞争，打破合作困境 | 头部高奖励，尾部低奖励 |
| 最低业绩门槛 | 消除"纯搭便车"策略 | 不达标无奖金 |
| 团队目标+个人分配 | 混合激励，团队达标才有分配 | 两层门槛 |
| 公开业绩榜 | 引入声誉机制 | 业绩透明化 |

## 案例 5：股权分配（创业团队）

### 局势描述
三人创业团队，如何分配股权：
- A：技术核心，产品开发
- B：市场销售，渠道资源
- C：财务运营，资金管理

### 合作博弈视角
三人合作创造价值100，任一人离开价值下降：
- A离开 → 价值降为40（无产品）
- B离开 → 价值降为60（无市场）
- C离开 → 价值降为70（无运营）

### Shapley值计算

**A的边际贡献**：
- 空联盟 → A加入：40-0=40
- B单独 → A加入：100-60=40（假设A+B能创造100）
- C单独 → A加入：40-0=40
- B+C → A加入：100-70=30

计算（简化）：
- A的平均边际贡献 ≈ (40+40+40+30)/4 = 37.5

同理计算B、C，得 Shapley值：
- A ≈ 37.5
- B ≈ 30
- C ≈ 20

### 核心检验
分配方案需满足：
- 每人单干收益 < 分配收益 → 无人愿离开
- A单干收益假设10 → 分配37.5 > 10 ✓
- B单干收益假设5 → 分配30 > 5 ✓
- C单干收益假设5 → 分配20 > 5 ✓

### 应用启示
- 核心"不可或缺"角色应获得更高比例
- 但需考虑实际贡献度而非理论值
- 动态调整机制：贡献变化 → 股权调整

## 案例 6：竞标博弈（政府项目招标）

### 局势描述
多个供应商竞标政府项目，中标者获得合同：
- 低价中标 → 获得合同但利润低
- 高价报价 → 利润高但中标概率低

### 拍卖博弈结构
相当于"密封投标第一价格拍卖"：
- 各方同时报价，信息保密
- 最高价（或最低价）者中标

### 策略分析
- 报价 = 成本 + 利润加价
- 利润加价越高 → 中标概率越低
- 最优报价平衡：利润加价 × 中标概率 最大化

### 信息不对称
- 各方成本不同（私人信息）
- 但可能知道成本分布
- 贝叶斯博弈分析

### 策略建议
| 信息状态 | 策略 |
|----------|------|
| 成本优势明显 | 适度加价，确保中标 |
| 成本劣势 | 激进报价争取机会，或放弃竞标 |
| 成本未知对手 | 按历史数据估计对手成本分布 |

## 案例 7：信任博弈（平台与商户）

### 局势描述
电商平台与商户：
- 商户可选择"诚信经营"或"欺骗消费者"
- 平台可选择"严格监管"或"宽松监管"

### 收益矩阵
```
        | 平台严格 | 平台宽松
商户诚信 | 50,30   | 50,60
商户欺骗 | 20,-10  | 80,20
```

### 分析
- 平台：宽松(60,20) > 严格(30,-10) → 倾向宽松
- 商户（平台宽松时）：欺骗(80) > 诚信(50) → 倾向欺骗
- 均衡：(欺骗,宽松)，收益(80,20)

### 问题
- 平台收益20低于严格监管时的30（如果商户诚信）
- 但商户诚信时平台宽松收益60更高
- 存在协调问题：需要商户先诚信才能证明严格监管的价值

### 改进策略
1. **引入惩罚**：欺骗被发现 → 罚款100
2. **改变收益**：
```
        | 平台严格 | 平台宽松
商户诚信 | 50,30   | 50,60
商户欺骗 | -30,-10 | -20,20（罚款后）
```
3. 新均衡：(诚信,宽松)，收益(50,60) ← 双方最优

### 应用启示
- 设计惩罚机制改变收益结构
- 惩罚成本需足够高才能遏制欺骗
- 平台监管成本需权衡

## 案例应用检查清单

分析商业问题时，依次检验：

1. **是否多方互动？** 单方决策不适用博弈论
2. **策略相互依赖？** 我的选择影响对方收益，反之亦然
3. **能识别经典博弈结构吗？** 对照囚徒困境、智猪博弈等模板
4. **信息是否对称？** 有私人信息 → 贝叶斯博弈
5. **是静态还是动态？** 有先后顺序 → 博弈树分析
6. **是一次还是重复？** 重复 → 合作可能
7. **收益结构如何？** 画出矩阵或树
8. **均衡是什么？** 预测结果
9. **如何改进？** 改变规则而非改变策略
